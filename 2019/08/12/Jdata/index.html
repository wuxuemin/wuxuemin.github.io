2. 特征

  1. 对离散值进行one-hot编码

  2. 对一段时间内的5种行为的次数进行统计（这里注意有些时间段的行为次数可能为0，例如行为5加入购物车在前面的一大段时间是没有的）计算的方法是在时间维度做了扩展，例如过去7天或者10天的次数统计，分别在过去的1, 2, 3, 5, 7, 10, 15, 21, 30天做了行为次数统计。

  3. 评论的特征做了进行离散化，再进行one-hot。将好坏评论的数目计算为好评率和差评率。

  1. 用户其他行为转化为购买行为的次数的比例（比较重要的特征）

3. 模型

  1. 训练集测试集划分：训练集的划分是将3.8-4.15的数据作为训练集，其中3.8-4.8的数据作为训练集数据特征获取的部分，4.9-4.15的数据作为标签构建。测试集使用3.15-4.15的数据作为测试集的特征获取部分的数据来源，预测未来的4.16-4.22的购买情况。
  2. 正负样本定义：期间发生购买行为的数据为正样本，其余为负样本。
  3. 我使用的是xgboost模型，80%的数据作为训练集，20%作为测试集进行模型的训练，考虑正负样本的不平衡，调整了参数scale_pos_weight
  4. 训练好的模型的输出概率中选取概率值大于0.7的视为正样本。（尝试了不同的值，0.7的效果是相对较好的）
  5. 最后的结果进行去重：用户对品类下的店铺只会购买一次。所以对同一用户id下的shop-id进行去重。

    4. 小tips

    由于数据量相对来说还是比较大的，所以在读入数据的时候进行了一个小的处理，减少数据占用内存的大小。（参考来自kaggle的处理方法）

5. 没尝试的点

  1. 对于时间的特征没有做好权重的区分，例如在时间靠前一点的特征相比较于时间较前的特征赋予更大的权重。
  2. 模型融合

    6. 从优秀者中学习到的点（以及本人的不足之处）

  3. EDA要做好，例如要先看下训练集的日期与行为的分布情况，由于一些特殊的日期（例如春节）可能会使得训练数据和测试数据分布不一致，所以要做一个事先的过滤。
  4. 构建时间相关的特征应注意剔除掉特殊的日期
  5. 特征的构造要考虑长期的特征（全部日期的统计数据）
  6. 在构造训练集和测试集中的一些考虑（可参考：https://zhuanlan.zhihu.com/p/71938382?utm_source=com.tencent.tim&utm_medium=social&utm_oi=555381879923224576 ）
